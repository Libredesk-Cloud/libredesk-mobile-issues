# Contributing to Libredesk Mobile Issues

Thank you for taking the time to contribute. This repository is the public issue tracker and release hub for Libredesk Mobile. Contributions here are focused on **issues, feedback, and documentation** — not application source code.

---

## Pull requests

This repository does not contain mobile app source code. **Pull requests are generally not expected.** The only acceptable PRs are updates to public documentation, issue templates, changelogs, or repository metadata.

If you have a code contribution to make to the app itself, please reach out via [SUPPORT.md](./SUPPORT.md) first to discuss the appropriate channel.

---

## How to write a useful bug report

A good bug report helps maintainers reproduce the issue quickly. Please use the Bug Report issue template and include:

- **App version** — visible in the app settings or About screen
- **Platform** — Android or iOS
- **Device model** — e.g., Samsung Galaxy S23, iPhone 14
- **OS version** — e.g., Android 14, iOS 17.4
- **Libredesk server version** — if known, available in your Libredesk admin panel
- **Installation source** — Google Play, App Store, GitHub Release APK, TestFlight, etc.
- **Steps to reproduce** — a numbered list of exact steps
- **Expected behavior** — what you expected to happen
- **Actual behavior** — what actually happened
- **Screenshots or screen recordings** — attach directly to the issue if applicable
- **Logs** — any error messages or crash logs you have access to

The more detail you provide, the faster the issue can be investigated.

---

## How to suggest a feature

Use the Feature Request issue template. When writing a feature request:

- Focus on the **problem you are trying to solve**, not just the solution
- Describe the **context** in which you would use this feature
- Mention the **platform** it applies to (Android, iOS, or both)
- If you have explored alternatives, mention what you considered

Feature requests are reviewed based on alignment with the product direction, feasibility, and demand from the user base.

---

## How to provide screenshots or screen recordings

You can drag and drop image or video files directly into the GitHub issue editor. For screen recordings, short clips (under 10 MB) work best. If the file is too large, consider uploading it to a third-party service and sharing the link.

When filing any issue, including the following context is always helpful:

|---|---|
| App version | 0.2.0 |
| Platform | Android |
| Device model | Pixel 7 |
| OS version | Android 14 |
| Libredesk version | 1.3.0 |
| Installation source | GitHub Release APK |

## Code of conduct

Please keep all interactions respectful and constructive. This project is maintained by a small team. Rude, dismissive, or demanding communication will not be tolerated and may result in issues being closed or users being blocked.

When in doubt, default to being kind and patient.
