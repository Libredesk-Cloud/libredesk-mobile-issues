# Security Policy

## Reporting a security vulnerability

**Do not report security vulnerabilities through public GitHub issues.**

If you believe you have found a security vulnerability in Libredesk Mobile, please disclose it responsibly by contacting the maintainers privately:

📧 **security@libredesk.cloud**

Please include as much detail as possible so the issue can be reproduced and assessed:

- A description of the vulnerability
- Steps to reproduce or a proof of concept
- The potential impact
- Your app version, platform, and server version if relevant

You will receive an acknowledgment within 5 business days. We ask that you allow a reasonable timeframe for the issue to be investigated and patched before any public disclosure.

---

## Examples of security issues

The following are examples of issues that should be reported privately rather than as public issues:

- Authentication bypass or session hijacking
- Unauthorized access to conversations, contacts, or agent data
- Token or credential leakage
- Sensitive data exposure in logs, local storage, or network traffic
- Authorization flaws (accessing data that should be restricted)
- Payment or subscription bypass (if applicable)
- Insecure data transmission

---

## Normal bugs

If you have found a bug that does not have a security impact, please open a regular [GitHub Issue](https://github.com/Libredesk-Cloud/libredesk-mobile-issues/issues/new/choose) using the Bug Report template.

---

## Scope

This security policy applies to the **Libredesk Mobile app** only. For vulnerabilities in the Libredesk server or core platform, please refer to the security policy of the upstream Libredesk project.
